# 架构说明 · 帮我说出口

本文档说明真实产品需要的架构，以及当前 H5 Demo 的简化方式。

## 一、当前 Demo 架构（初赛阶段）

### 1.1 技术栈

- Vite + React + TypeScript
- 纯静态前端，无后端、无数据库、无登录、无外部 API
- 本地规则和模板模拟受控 AI 转译

### 1.2 简化方式

Demo 用**模块级内存状态**模拟三方收件箱与会话存储：

- `src/lib/messageStore.ts` 维护家长端和孩子端两个内存数组
- 同一设备切换身份即可看到消息流（模拟跨用户）
- 刷新即销毁，符合隐私最小化
- 提供"加载演示案例"按钮，方便评委快速查看家长端效果

### 1.3 Demo 闭环

1. 孩子端：倾诉 → 安全确认（若命中）→ 澄清三区块 → 选分享范围 → 预览草稿 → 确认发送
2. 家长端：收件箱看到沟通请求（温和摘要，非原话）→ 问 AI 怎么回应 → AI 通用建议（含防火墙说明）→ 回应草稿 → 确认发送
3. 孩子端：收件箱收到家长回应
4. 老师端：独立流程，记录观察 → 谈话建议+观察点+转介

### 1.4 Demo 局限

- 无真实跨设备同步
- 无真实用户身份与关系
- 无真实 LLM，所有建议为预审模板
- 无真实消息通知
- 无持久化，刷新即销毁

## 二、真实产品架构

### 2.1 后端服务

- **用户身份与关系**：孩子、家长、老师三种身份；监护关系/班级关系绑定
- **会话存储**：三方各自独立会话，会话内容加密存储
- **分享授权记录**：记录每条消息的分享范围、授权时间、授权对象
- **消息通知**：新消息提醒（推送/短信/微信，需用户同意）
- **日志与删除机制**：操作日志可审计，敏感数据支持按需删除

### 2.2 LLM 服务

- 受安全规则约束的 LLM
- 输出在返回前端前过审，拦截诊断词/严重度词/危险方法
- 严格遵守信息防火墙，不泄露未授权原话
- 单次生成即结束，禁止长期陪聊

### 2.3 权限控制

- 谁能看哪条消息由"分享授权"决定，不是默认开放
- 孩子原话默认仅孩子可见
- 家长/老师只能看到孩子"确认分享"的对外消息
- "模糊提醒"模式下，家长端只看到"孩子想跟你聊聊"

### 2.4 信息防火墙（后端实现）

- 原话默认不共享
- 具体事实默认不共享
- 只共享用户明确允许的内容
- 未授权时只给对方通用沟通建议
- 分享前必须让用户确认
- 对方不能通过追问套出孩子隐私
- AI 不能说"孩子其实告诉我……"

### 2.5 输出安全审查

- LLM 输出在返回前端前过审
- 拦截：诊断词（抑郁/焦虑/双相/ADHD 等）、严重度词（轻度/中度/重度/危机）、治愈承诺词、危险行为方法
- 高风险表达命中即进安全确认流程，不进入 LLM 生成

### 2.6 数据留存与删除

- 敏感输入最小化留存
- "暂时不想分享"内容不落库
- 支持用户主动删除
- 符合《儿童个人信息网络保护规定》《未成年人网络保护条例》

### 2.7 合规

- 《儿童个人信息网络保护规定》：正当必要、知情同意、目的明确、安全保障
- 《未成年人网络保护条例》：最有利于未成年人原则、网络保护义务
- UNICEF 儿童 AI 指南：隐私、安全、透明、儿童最佳利益

## 三、从 Demo 到真实产品的演进路径

1. **后端化**：把 messageStore 内存状态迁移到后端会话存储 + 数据库
2. **身份系统**：引入用户身份与监护/班级关系绑定
3. **LLM 接入**：替换本地模板为受安全规则约束的 LLM，输出经审查后返回
4. **消息通知**：接入推送/短信/微信通知（需用户同意）
5. **权限细化**：分享授权记录可审计，支持撤回
6. **持久化与删除**：敏感数据支持按需删除，符合合规要求
7. **多端同步**：支持跨设备查看消息
