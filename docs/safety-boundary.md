# 安全边界 · 帮我说出口

本文档固化产品的安全边界，所有生成逻辑、界面文案、测试用例都必须符合本文档。

## 一、产品边界（绝对不做）

1. 不做心理诊断。
2. 不输出抑郁症、焦虑症等病名作为判断或标签。
3. 不做心理测评或分数评估。
4. 不判断严重程度。
5. 不承诺治疗效果。
6. 不替代心理老师、心理咨询师、医生或学校已有流程。
7. 不做长期 AI 陪聊。
8. 不收集真实姓名、学校、班级、手机号。
9. 不保存敏感输入。
10. 不自动泄露任何一方原话。

## 二、术语边界说明

"抑郁""焦虑"等词的使用边界：

- **可以出现**：在本安全边界文档、项目背景文档、界面边界声明中，用来解释"本工具不做诊断"。
- **不可以出现**：在生成给用户的对外消息、AI 建议中，作为判断或标签出现。

## 三、安全确认（替代旧"高风险判定"）

### 3.1 设计原则

"高风险表达"不等于系统判定孩子有危险，但也不能直接放行继续整理。

命中相关表达后进入"安全确认页"，由用户自己选择当前状态。

### 3.2 触发词列表

想死、不想活、自杀、自残、伤害自己、伤害别人、结束一切、永远消失、活着没意思、撑不下去、想跳楼、想割腕，以及常见变体（不想活了、寻死、轻生、了结自己、活不下去、撑不住、不想存在、消失算了等）。

### 3.3 检测策略

- 输入预处理：去空格、转小写、繁简统一。
- 关键词子串匹配 + 少量正则覆盖变体。
- 命中即返回需要安全确认，不返回具体命中的词给前端展示。

### 3.4 安全确认页两路分支

- **分支 A（我现在安全，只是想整理表达）**：承认这可能只是情绪化表达，用户可选择继续整理流程。
- **分支 B（我不太安全 / 我身边马上有危险）**：不进入普通整理流程，只显示可信成年人、学校心理老师、12355/12356、110/120 等支持资源。

### 3.5 安全确认页文案规范

- 承认：这可能只是情绪化表达，也可能真的需要帮助。两种情况都正常。
- 不询问危险方式、地点、计划、时间等细节。
- 不输出任何危险行为方法。
- 资源分层：
  - 心理支持与求助咨询：12355 青少年服务热线、12356 心理援助热线、可信成年人/学校心理老师
  - 如存在立即人身危险：110 / 120

## 四、信息防火墙

### 4.1 核心规则

1. 原话默认不共享。
2. 具体事实默认不共享。
3. 只共享用户明确允许的内容。
4. 未授权时只给对方通用沟通建议。
5. 分享前必须让用户确认。
6. 对方不能通过追问套出孩子隐私。
7. AI 不能说"孩子其实告诉我……"。

### 4.2 家长端防火墙

- AI 可以给破冰建议，但不能根据孩子未授权的具体事实做过度定制建议。
- 如果孩子没有授权分享具体内容，家长端只能得到通用沟通建议，例如"先不要追问具体原因，可以先表达你愿意听"。
- 不能暗示孩子具体说了什么、经历了什么。
- 家长追问孩子具体内容时，AI 必须拒绝并给通用建议。

### 4.3 老师端防火墙

- 老师记录的观察不直接变成对家长的告知。
- 老师端独立流程，不接收孩子消息（MVP 阶段）。
- 不自动告知家长或其他老师。

### 4.4 分享授权

- 孩子选择分享范围后才生成对外消息草稿。
- 对外消息必须经"预览 + 确认"两步。
- "模糊提醒"模式下，家长端只看到"孩子想跟你聊聊"，看不到任何具体内容。

## 五、生成层安全

- 卡片文案来自预审模板，初赛阶段不调用 LLM，用本地规则和模板模拟受控 AI 转译。
- 模板中禁止出现：作为判断或标签的诊断词、严重度词、承诺治愈词。
- 模板中禁止出现：危险行为方法、对家长的指责性表述。
- 用户输入仅用于"澄清"环节，澄清内容默认私密，不进入对外消息（除非孩子授权）。

## 六、隐私边界

- 用户需要登录，系统有后端和数据库（MySQL），所有敏感内容使用 AES-256-GCM 加密存储。
- 应用内不主动收集真实姓名、学校、班级、手机号等身份信息（display_name 仅为展示用昵称）。
- 用户输入加密存储于后端数据库，仅用于生成降敏摘要和工作记忆，不对外暴露原文。
- 孩子的消息默认 L0 级别（完全私密），仅孩子本人和 AI 可读取；对外分享需要孩子明确确认。
- 所有敏感操作（消息发送、分享确认、撤回、记忆删除等）都有审计日志记录。
- 复制功能走浏览器原生 Clipboard API，不经过额外服务端中转。

## 七、合规依据

- 《儿童个人信息网络保护规定》：正当必要、知情同意、目的明确、安全保障。
- 《未成年人网络保护条例》：最有利于未成年人原则、网络保护义务。
- UNICEF 儿童 AI 指南：隐私、安全、透明、儿童最佳利益。

## 八、界面常驻边界声明

每个页面底部固定：本工具不做心理诊断、不替代专业人员；需要心理支持可拨打 12355 / 12356；如存在立即人身危险请拨打 110 / 120。AI 是受托的中间人，不会自动泄露任何一方原话；所有对外消息必须经你确认后才分享。

## 九、LLM 接入约束

系统支持接入 OpenAI 兼容的外部 LLM API。当配置了 LLM 时，用户输入会发送到外部 LLM 服务用于生成降敏摘要。未配置 LLM 时，系统使用本地模板 fallback，核心功能不受影响。

接入 LLM 时必须满足以下约束：

1. 先做安全确认（本文件第三节），命中高风险关键词即进安全确认页，不调用 LLM。
2. LLM 输出必须经过非诊断审查，不得出现病名标签、严重度词、治愈承诺。
3. 严格遵守信息防火墙（本文件第四节），不泄露未授权原话。
4. 禁止长期陪聊和多轮对话，单次生成即结束。
5. 不得输出危险行为方法。
6. 孩子原文进入 LLM 前，必须经过安全检测；LLM 仅用于生成降敏摘要，不直接转发原文给任何第三方。
7. 家长/老师向 AI 提问时，AI 不接收孩子未授权的具体内容，只返回通用沟通建议。
8. 所有 LLM 调用都有审计日志记录（记录调用来源和返回来源，不记录原文）。
9. 如需完全不上传用户输入到外部服务，可不配置 LLM_API_KEY，系统将使用本地模板。
